CSP-垦田计划

2023-03-29 C/C++, CSP, Python 0 评论
  • 提交次数：2
  • 得分：70 – 100

问题

  • http://118.190.20.162/view.page?gpid=T164

方法

第一次提交

一开始想到用优先队列/最大堆，在读取输入的时候挨个插入形成最大堆，然后从顶部开始挨个拿出，判断：

  • 所剩时间是否已经小于或等于 k
  • 所剩资源是否不足以减少
    • 如果不足，那么意味着当前的所剩时间，就是题目要求的全部时间（因为是大根堆从大到小拿的，并且全部时间只跟最大剩余时间有关）
    • 如果充足，则将所剩时间和所剩资源都相应减少，然后再插入到大根堆中

很明显这样的时间复杂度难以应对 10^9 的数据量，于是第一次提交得分 70，显示运行超时。

第二次提交

抓住一个关键：全部时间只跟最大剩余时间有关，也就意味着只有当手里的资源足以让耗时最多的那些耗时相同的计划都减少时间，那么才对最终结果有影响，所以可以将所有耗时相同的计划合并成一个整体来看，那么其消耗的资源就是他们分别消耗的资源的总和。

可以采用 map 来维护这样的一张耗时+消耗资源表，就以其耗时作为键，并且再维护一个 max 变量来记录当前耗时最多的项目的耗时。然后开启循环，从 max 开始遍历这张表，存在三种情况：

  1. max 已经不大于 k：直接退出循环，输出 max
  2. m 资源不足以将此次遍历到的耗时减少：退出循环，输出 max
    • 因为该 map 是以耗时作为键的，而且我们是从最大的耗时开始遍历，那么当资源不足以支撑此次消耗时，那么意味着存在那么一个项目，无法被削减时间，那么总时间就会是这个没有被削减的最大时间了。
  3. 以上两种情况都不成立，则：
    • 资源减少相应值
    • map 的下一次遍历到的数据中，预期资源消耗量加上本次遍历到的资源消耗量（相当于本次的资源给出去后，较大的时间消耗都回退了 1 单位时间）
    • 更新 max 为当前遍历的 key （因为是以耗时作为键的，max 始终记录当前最大耗时）

Tips：数据量为 10^9，需要使用 long long int
