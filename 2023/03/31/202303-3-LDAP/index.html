202303-3 LDAP

2023-03-31 CSP C++ 评论
  • 提交次数：2
  • 得分：70（运行超时）– 100

问题

  • http://118.190.20.162/view.page?gpid=T163

方法

第一次提交

  • 模拟

  • 使用 map<int, map<int, int>> 来记录用户和其属性键值对，使用一个备忘录 map<int, vector<int>> 来记录拥有某属性的用户 id

  • 使用一个操作符栈、一个数字栈、一个集合栈，分别记录：

    • 操作符：&/|/:/~
    • 数字
    • 某一子表达式的结果

  • 对于每一个表达式，遍历其中每一个字符：

    • 可以保证不会连续出现两个左括号，因为指令的格式为 [&/|]([属性][~/:][值])，即使是非常复杂的嵌套，也无法连续出现两个左括号，所以，当遇到 ( 时，直接忽略，遍历下一个字符。

    • 将数字字符转换为 int 后压入数字栈。

    • 将无论是 &/|/:/~ 的操作符，一旦遍历到就压入操作符栈。

    • 一旦遍历到右括号 )，或是碰到了表达式的边界，则立刻执行出栈操作：

      • 首先将操作符栈取栈顶后出栈，若该操作符 op 为 & 或 |，那么意味着数字栈是空的

        这是因为这两个操作符 &/| 只有可能操作若干个子表达式的结果，也就是两个集合，而不可能直接操作数字栈里的东西。

        对于集合栈进行连续两次出栈，取出两个集合，对应操作符进行交集或并集运算，然后将结果压入集合栈

      • 若取出的操作符 op 为 : 或 ~，那么对于数字栈进行连续两次出栈，取出两个数字

        特别注意的是： 先取出的数字是值 value，后取出的数字是键 key

        然后对应操作符进行寻找，也就是寻找属性名为 key，属性值为 value 的用户 id，具体操作为：

        1. 在上面提到的备忘录中，取出拥有属性名 key 的所有用户的 id 列表
        2. 遍历这个用户列表，将符合条件的 id 挨个插入 set<int> 然后返回

  • 执行完遍历后，我们就可以拿到一个仅仅只有1个元素的集合栈了，出栈后返回，将该元素内的所有用户 id 按要求输出。

最终得分 70。

100 的测试点显示运行超时，目前时间复杂度瓶颈在交集上，猜测可以优化一下集合的数据结构或者新增一个结果备忘录…

第二次提交

  • 在第一次提交的基础上，将 set<int> 来记录 & 与 | 运算的结果改为了 bitset<2500>，方便在计算交集和并集时采用位运算飞快计算。

    题目给出了数据限制，至多只存在 2500 个用户

  • 相比起第一次提交，新增一个 int[2500] 的数组，为的是存储第 i 个输入的用户编号，用以对应 bitset 中的每一位。

  • 输入时维护的备忘录更加细分类别，采用数据结构为 map<int, map<int, vector<int>>>，将相同 属性编号 和相同 属性值 的用户归为一类，方便在进行 : 操作时快速将符合条件的用户找到。但于此同时，就需要将 ~ 操作的情况分列开，遍历具备某一属性编号的所有值–用户ID列表的map。
